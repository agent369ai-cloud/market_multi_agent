# Contributing to Market Merchant Support Assistant

Thank you for your interest in contributing to the Market Merchant Support Assistant project.

## How to contribute

1. Fork the repository.
2. Create a new branch for your feature or fix:

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes.
4. Install and run tests locally if applicable.
5. Commit your changes with a clear message.
6. Push your branch to your fork and open a pull request.

## Code style

- Keep Python code readable and consistent.
- Use descriptive function and variable names.
- Add comments when the behavior is not obvious.

## Issues and suggestions

- If you find a bug or want to propose an improvement, open an issue first.
- Provide steps to reproduce and expected vs actual behavior when reporting bugs.

## Pull request checklist

- [ ] Code changes are small and focused.
- [ ] The README is updated if the change affects usage or setup.
- [ ] No sensitive data is included in the commit.
- [ ] The project still runs after your change.
